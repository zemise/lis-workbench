# 检验结果趋势图开发计划

## 目标

在当前检验结果查询工具中，支持查询某个患者在特定检验仪器下的历史结果，并将同一项目按时间生成趋势图，方便观察项目变化。

第一阶段优先保证数据正确、查询稳定、结果可核对，不追求复杂图表效果。

## 当前实现状态

已完成第一阶段基础版本：

- 新增 `src/trend_core.h/.cpp`。
- 新增 `src/trend_window.h/.cpp`。
- 新增 `src/trend_chart_renderer.h/.cpp`。
- 主界面底部新增 `趋势图` 按钮。
- 趋势查询复用当前左侧查询条件。
- 趋势查询要求至少一个患者相关条件，并要求开始/结束日期。
- 趋势窗口支持项目下拉、简单折线图和趋势明细表。

已完成第一批 SCI 风格图表优化：

- 图表绘制逻辑从窗口层拆到 `trend_chart_renderer.*`。
- 横轴按有效结果点顺序等距绘制，标签显示真实 `REP_DATE`。
- 纵轴按结果值和参考区间动态计算范围。
- 支持参考区间浅灰背景带和上下限虚线。
- 支持按 `NORMAL` 码值给异常点着色。
- 补充图表标题、坐标轴标题、网格线和图例。

仍需现场验证：

- `REP_DATE` 是否完全符合现场趋势时间预期。
- 单患者条件是否足够精确，是否需要强制诊疗卡号/病人号/条码号。
- 大日期范围下查询耗时是否可接受。
- `NORMAL` 码值颜色在趋势图中是否符合现场观察。

## 推荐范围

第一阶段只做：

- 单患者。
- 单检验仪器。
- 日期范围。
- 单项目趋势图。
- 趋势数据表格同步展示。
- 只绘制可转换为数值的结果。

暂不做：

- 多患者对比。
- 多仪器混合趋势。
- 多单位项目叠加在同一坐标轴。
- 非数值结果画图。
- 复杂统计分析。

## 数据关系

核心表关系：

```text
LS_AS_REPORT.REP_NO = LS_AS_REPENTRY.REP_NO
LS_AS_REPENTRY.ITEM_CODE = LS_AS_ITEM.ITEM_CODE
LS_AS_REPORT.MACH_CODE = LS_AS_MACHINE.MACH_CODE
```

患者定位沿用当前主界面已有条件：

```text
LS_AS_REPORT.REG_NO
LS_AS_REPORT.TXM_NO
LS_AS_REPORT.NAME
LS_AS_BARCODE.REG_NO
LS_AS_BARCODE.BARCODE = LS_AS_REPORT.TXM_NO
```

趋势时间优先使用：

```text
LS_AS_REPORT.REP_DATE
```

如果现场确认 `REP_DATE` 不稳定，再评估改为 `CHK_DATE`。当前主界面“上机时间”也已经按 `REP_DATE` 展示。

## 趋势数据字段

趋势查询至少返回：

| 字段 | 来源 |
| --- | --- |
| 报告号 | `LS_AS_REPORT.REP_NO` |
| 条码号 | `LS_AS_REPORT.TXM_NO` |
| 样本号 | `LS_AS_REPORT.OPER_NO` |
| 患者姓名 | `LS_AS_REPORT.NAME` |
| 仪器代码 | `LS_AS_REPORT.MACH_CODE` |
| 报告时间 | `LS_AS_REPORT.REP_DATE` |
| 项目代码 | `LS_AS_REPENTRY.ITEM_CODE` |
| 项目名称 | `LS_AS_ITEM.ITEM_NAME` |
| 英文名 | `LS_AS_ITEM.ENG_NAME` |
| 结果原文 | `LS_AS_REPENTRY.RESULT` |
| 数值结果 | 程序从 `RESULT` 转换 |
| 单位 | `LS_AS_ITEM.UNIT` |
| 下限 | `LS_AS_REPENTRY.UPBOUND` |
| 上限 | `LS_AS_REPENTRY.DOWNBOUND` |
| 异常标记 | `LS_AS_REPENTRY.NORMAL` |

注意：当前项目已确认“上限对应 `DOWNBOUND`，下限对应 `UPBOUND`”。

## 查询策略

第一阶段建议新增独立查询函数，不影响现有主查询：

```text
trend_core.h/.cpp
```

建议结构：

```cpp
struct TrendPoint {
    std::string rep_no;
    std::string txm_no;
    std::string oper_no;
    std::string patient_name;
    std::string report_time;
    std::string item_code;
    std::string item_name;
    std::string item_eng;
    std::string result_text;
    double result_value;
    bool has_numeric_value;
    std::string unit;
    std::string lower_bound;
    std::string upper_bound;
    std::string normal;
};
```

SQL 查询原则：

- 必须带日期范围，避免全库扫表。
- 必须带患者条件之一，避免误查大量患者。
- 如果指定仪器，则必须按 `LS_AS_REPORT.MACH_CODE` 过滤。
- 如果指定项目，则按 `LS_AS_REPENTRY.ITEM_CODE` 过滤。
- 排序按 `REP_DATE ASC, REP_NO ASC`。

## UI 方案

第一阶段新增一个独立窗口，不挤占当前主界面：

```text
trend_window.h/.cpp
```

入口建议：

- 当前主界面底部新增 `趋势图` 按钮。
- 默认带入当前左侧查询条件。
- 如果中间列表已选中某条报告，可默认带入该患者和仪器。

趋势窗口布局：

- 顶部：患者摘要、仪器、日期范围、项目选择。
- 左侧或上方：项目列表。
- 中间：折线图。
- 下方：趋势数据表格。

第一阶段可以先用 Win32 GDI 画简单折线：

- 横轴：有效结果点顺序，标签显示真实检测日期/时间。
- 纵轴：数值结果。
- 点：每次报告结果。
- 悬浮提示暂不做。
- 参考范围背景带可留到第二阶段。

## 图表规则

- 只画 `RESULT` 能安全转换成数字的点。
- 非数值结果仍显示在趋势表格，但不参与折线。
- 同一项目单位变化时，默认不合并画图，需要在表格中提示。
- 多项目趋势后续再做，第一阶段只允许一个项目一张图。
- 相同时间多条结果按 `REP_NO` 排序展示，不自动去重。

## SCI 风格图表优化标准

当前第一阶段图表只解决“能看趋势”。如果后续目标是接近论文、汇报或科室质控图的效果，需要按以下标准继续优化。

### 数据可信度

- 横轴默认按有效结果点顺序等距绘制，标签必须显示真实检测日期/时间。
- 如后续需要表达检测间隔密度，可增加“真实时间轴”切换项，但默认不启用。
- 纵轴刻度需要根据项目结果范围和小数精度动态生成，避免过粗或过密。
- 点位需要能追溯原始数据，至少包含检测时间、结果、单位、报告号、项目名称。
- 图上应标出参考区间，可采用浅灰背景带或上下限虚线。
- 异常点应和 `NORMAL` 规则保持一致，高值红色、低值蓝色、正常深色。
- 同一项目如果单位发生变化，默认禁止合并绘图，并在界面提示“单位不一致”。
- 原始结果文本和数值转换结果都要保留，避免因格式化造成临床含义变化。

### 视觉规范

- 使用白色背景、浅灰网格线、黑色坐标轴，避免过重的界面控件感。
- 折线使用单一主色，点位使用高对比颜色区分正常、高值、低值。
- 标题建议格式为：`项目名称（英文名，单位）趋势图`。
- 横轴标题固定为 `检测日期（按结果顺序等距）`，纵轴标题格式为 `结果值 (单位)`。
- 图例应说明结果线、参考范围、高值点、低值点，位置优先右上角或底部。
- 图表边距要给足，避免坐标轴文字、标题、图例被裁切。
- 日期标签需要自动避让，点数较多时减少横轴标签数量。

### 导出质量

- 支持导出 PNG，分辨率建议不低于 `1600x1000`。
- 如果用于论文或正式汇报，优先支持 SVG 或 PDF 矢量导出。
- 导出图不能直接截图控件，应使用同一套绘图逻辑做离屏重绘。
- 导出时应隐藏调试控件和窗口边框，只保留图表、标题、坐标轴、图例和必要注释。
- 需要保留导出时的项目、患者、仪器、日期范围等元信息，方便后续追溯。

### 交互与功能

- 支持快捷时间范围，例如近 7 天、30 天、90 天、全年。
- 支持点击图中点位后，高亮下方趋势表格对应行。
- 支持从趋势表格反向定位图中点位。
- 多项目趋势优先采用分面图，不同单位项目不要强行叠加到同一坐标轴。
- 同单位项目可以允许叠加，但必须显示清晰图例。
- 平滑曲线只能作为可选显示，默认必须使用真实折线，避免误导结果变化。

### 技术拆分建议

建议第二阶段把图表绘制从窗口逻辑中拆出独立渲染层：

```text
trend_core.*             数据查询和趋势点结构
trend_window.*           Win32 窗口、控件、事件
trend_chart_renderer.*   坐标换算、网格、参考区间、折线、点、标题、导出
```

短期仍可继续使用 Win32 GDI，但需要把绘图逻辑集中到 `trend_chart_renderer.*`，避免后续替换 Qt Charts 或 QCustomPlot 时牵动窗口和数据库层。

### 实施优先级

1. 结果点等距横轴、动态纵轴、参考区间背景带。
2. 高低异常点颜色、图例、完整坐标轴标题。
3. 图点和趋势表格联动。
4. PNG 高分辨率导出。
5. SVG/PDF 矢量导出。
6. 多项目分面图和同单位叠加图。

## 开发步骤

1. 新增趋势数据结构和 SQL 查询。
2. 先用日志或临时表格验证查询结果是否和主界面一致。
3. 新增趋势窗口，只显示趋势表格。
4. 增加单项目选择。
5. 增加 GDI 折线图。
6. 增加主界面 `趋势图` 按钮入口。
7. 实机验证同一患者、同一仪器、同一项目的历史趋势。
8. 根据现场反馈决定是否增加多项目、多单位、参考范围背景带。

## 验证重点

- 患者过滤是否准确。
- 仪器过滤是否准确。
- `REP_DATE` 时间排序是否符合现场观察。
- `RESULT` 数值转换是否稳定。
- 单位是否来自 `LS_AS_ITEM.UNIT`。
- 趋势表格和主界面右侧明细是否能互相核对。
- 大日期范围下查询是否会明显卡顿。

## 风险点

- 同名患者可能很多，必须尽量使用诊疗卡号、病人号或条码号缩小范围。
- `RESULT` 可能包含非数字、符号或文本，需要容错。
- 同一项目不同仪器的结果不一定能直接比较，因此第一阶段限定单仪器。
- 同一项目单位变化时，不能直接画在同一条折线上。
- 日期范围过大时可能对 LIS 数据库造成压力，需要保留明确日期过滤。
